# Invariant-Core - Foundations

Foundations define the base concepts used by the invariant system before they are organized into layers, glossary terms, or canonical invariant IDs.

---

## Documents

* [Pressure](Pressure.md)
  The unresolved possibility carried by a constrained system before validation, elimination, leakage, or closure.

---

## Role

Foundation documents describe the primitives that the rest of `Invariant-Core` depends on:

* how constrained systems carry unresolved burden
* how that burden moves through boundaries
* how invalidity is resolved or expelled
* how closure becomes possible
