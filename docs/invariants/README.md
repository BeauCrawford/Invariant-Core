# Invariant-Core - Invariants

Invariants define the conditions that valid states, transitions, and compositions must preserve across the system.

---

## Documents

* [Invariant Layers](Invariant-Layers.md)
  The descending authority stack that organizes constraint strength, pressure exits, seals, and enforcement.

* [Core Invariants](core/README.md)
  The canonical invariant set for structure, boundary enforcement, validity, pressure resolution, closure, and truth.

---

## Role

Invariant documents specify the rules that foundation concepts must satisfy:

* what valid structure requires
* where constraints become authoritative
* how invalidity is prevented from surviving
* how pressure resolves into closure
* how lower-layer invariants bind higher-layer systems
